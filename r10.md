## R9 Critical Issues — Verification Status

### ✅ R9-C01: Command Injection via `execute_command_safe()` — **FIXED**

| Property | Value |
|----------|-------|
| **File** | `src/core/src/spoofer.cpp` |
| **Lines** | 87-95 |
| **Status** | ✅ **RESOLVED** |
| **Fix Quality** | Excellent |

**Original Issue**: Command string concatenation allowed argument injection via crafted adapter names.

**Fix Applied**:
```cpp
// R9-C01: Each argument is quoted to prevent injection via spaces
// or special characters in adapter names like "Wi-Fi & calc.exe".
std::string cmd_line = command;
for (const auto& arg : args) {
    // Quote each argument to prevent argument injection
    cmd_line += " \"" + arg + "\"";
}
```

**Verification**: ✅ Each argument is now quoted. Combined with `is_safe_argument()` validation (which rejects `;|&$`'\\<>(){}[]!#~`), this prevents argument injection attacks.

**Remaining Concern**: ⚠️ **Still uses string concatenation** instead of proper argument arrays. While quotes mitigate injection, a more robust solution would use `CreateProcessW` with `argv`-style array or direct Windows API calls (`SetDnsServers()`).

**Recommendation**: Consider migrating to Windows API for critical operations (DNS, interface configuration) to eliminate command-line parsing entirely.

### ✅ R9-C02: Unsafe Memory Wipe in `safe_wipe()` — **FIXED**

| Property | Value |
|----------|-------|
| **File** | `src/core/src/ncp_paranoid.cpp` |
| **Lines** | 165-215 |
| **Status** | ✅ **RESOLVED** |
| **Fix Quality** | Excellent |

**Original Issue**: `hop_chains.clear()` did not call destructors on elements, leaving key material in heap.

**Fix Applied**:
```cpp
void safe_wipe() {
    // ... clear circuits and bridge nodes ...

    // R9-C02: Wipe hop chain key material with proper shrink_to_fit
    for (auto& cs : hop_chains) {
        for (auto& km : cs.key_material) {
            sodium_memzero(km.shared_key, sizeof(km.shared_key));
            sodium_memzero(km.layer_nonce, sizeof(km.layer_nonce));
        }
        cs.key_material.clear();
        cs.key_material.shrink_to_fit();  // ← Added
    }
    hop_chains.clear();
    hop_chains.shrink_to_fit();  // ← Added

    // Also wipe RAM-only path
    if (!ram_only_base_dir.empty()) {
        sodium_memzero(&ram_only_base_dir[0], ram_only_base_dir.size());
    }
    ram_only_base_dir.clear();
    ram_only_base_dir.shrink_to_fit();
    // ...
}
```

**Verification**: ✅ All vectors now call `shrink_to_fit()` after `clear()`, forcing deallocation of internal buffers. Key material is explicitly zeroed before clearing.

**Additional Improvement**: The destructor now calls `safe_wipe()` instead of `sodium_memzero(impl_.get(), sizeof(Impl))`, avoiding UB from zeroing live C++ objects with vtables.

### ✅ R9-C04: TLS Session Key Rotation Race Condition — **FIXED**

| Property | Value |
|----------|-------|
| **File** | `src/core/src/mimicry.cpp` |
| **Lines** | 135-163 |
| **Status** | ✅ **RESOLVED** |
| **Fix Quality** | Excellent |

**Original Issue**: Counter updated with `memory_order_relaxed`, allowing concurrent threads to read stale values.

**Fix Applied**:
```cpp
void TrafficMimicry::set_tls_session_key(const std::vector<uint8_t>& key) {
    // ...
    // R9-C04: Use seq_cst for strongest ordering guarantee
    tls_packets_since_key_rotation_.store(0, std::memory_order_seq_cst);
}

void TrafficMimicry::rotate_tls_session_key() {
    std::lock_guard<std::mutex> lock(tls_key_mutex_);
    // ...
    // R9-C04: Use seq_cst for strongest ordering guarantee
    tls_packets_since_key_rotation_.store(0, std::memory_order_seq_cst);
}

// In wrap_payload():
// R9-C04: Use acq_rel for proper synchronization with rotate_tls_session_key()
if (tls_packets_since_key_rotation_.fetch_add(1, std::memory_order_acq_rel) 
    >= config_.tls_key_rotation_packets) {
    rotate_tls_session_key();  // Already holds lock
}
```

**Verification**: ✅ 
- `store()` uses `memory_order_seq_cst` (strongest ordering)
- `fetch_add()` uses `memory_order_acq_rel` (acquire-release semantics)
- This ensures proper synchronization: rotation happens-before packet encryption

## New Critical Issues (R10)

### 🔴 R10-C01: Incomplete Certificate Validation in `CertificatePinner`

| Property | Value |
|----------|-------|
| **File** | `src/core/src/security.cpp` |
| **Severity** | 🔴 Critical (CWE-295) |
| **CVSS Score** | 7.5 (High) |
| **Status** | ⬜ Open |

**Description**:
The `CertificatePinner::verify_certificate()` function checks pin matching but **does not validate certificate chain depth or intermediate CA authenticity**.

**Code Pattern** (from AUDIT.md and R9-H01):
```cpp
bool CertificatePinner::verify_certificate(const std::string& hostname,
                                            const std::string& cert_hash) const {
    // Only checks if cert_hash matches pinned hash
    // Missing: chain depth validation, intermediate CA verification,
    //          expiry check, revocation check (CRL/OCSP)
}
```

**Impact**: 
- Expired certificates may be accepted if hash matches
- Revoked certificates (e.g., compromised CA) may be accepted
- Self-signed certificates with matching hash are accepted (no chain validation)

**Recommendation**:
```cpp
bool CertificatePinner::verify_certificate(SSL* ssl, const std::string& hostname) {
    X509* cert = SSL_get_peer_certificate(ssl);
    if (!cert) return false;

    // 1. Check expiry
    if (X509_cmp_current_time(X509_get_notBefore(cert)) >= 0 ||
        X509_cmp_current_time(X509_get_notAfter(cert)) <= 0) {
        X509_free(cert);
        return false;
    }

    // 2. Check revocation (CRL/OCSP)
    if (is_revoked(cert)) {
        X509_free(cert);
        return false;
    }

    // 3. Check chain depth
    STACK_OF(X509)* chain = SSL_get_peer_cert_chain(ssl);
    if (sk_X509_num(chain) > kMaxChainDepth) {
        X509_free(cert);
        return false;
    }

    // 4. Check pin
    std::string hash = compute_cert_hash(cert);
    X509_free(cert);
    return is_pinned(hostname, hash);
}
```

### 🔴 R10-C02: Hardcoded DoH Servers Without Fallback

| Property | Value |
|----------|-------|
| **File** | `src/core/include/ncp_spoofer.hpp` |
| **Severity** | 🔴 Critical (CWE-693: Protection Mechanism Failure) |
| **CVSS Score** | 6.5 (Medium) |
| **Status** | ⬜ Open |

**Description**:
Default DoH servers are hardcoded without user configuration or regional fallbacks:

```cpp
doh_servers = {
    "https://1.1.1.1/dns-query",
    "https://8.8.8.8/dns-query",
    "https://9.9.9.9/dns-query"
};
```

**Problems**:
1. **No certificate pinning** for DoH servers (R9-H03)
2. **No fallback** if all three are blocked (e.g., national firewalls in Russia, China, Iran)
3. **No user configuration** option to add custom DoH servers
4. **No regional fallbacks** (e.g., Yandex DNS `https://dns.yandex.net/dns-query` for Russia)

**Impact**: 
- Complete DNS failure in regions blocking Cloudflare/Google/Quad9
- No workaround for users without code modification

**Recommendation**:
```cpp
struct DoHConfig {
    std::vector<std::string> servers = {
        "https://1.1.1.1/dns-query",
        "https://8.8.8.8/dns-query",
        "https://9.9.9.9/dns-query",
        // Regional fallbacks
        "https://dns.yandex.net/dns-query",      // Russia
        "https://doh.dns.sb/dns-query",          // Global (DNS.SB)
        "https://dns.adguard.com/dns-query"      // Alternative
    };
    std::vector<std::string> pinned_hashes;      // Certificate pins
    bool allow_user_override = true;
};
```
