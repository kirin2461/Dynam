## Critical Issues (R13)

### 🔴 R13-C01: DNS Response Parsing — Buffer Over-read

| Property | Value |
|----------|-------|
| **File** | `src/core/src/doh.cpp` |
| **Lines** | 340, 957, 1455 |
| **Severity** | 🔴 **CRITICAL** (CWE-125: Out-of-bounds Read) |
| **CVSS** | 7.5 (High) |
| **Status** | ⬜ Open |

**Vulnerable Code**:
```cpp
// doh.cpp:340 — parse_dns_response()
} else if (rtype == static_cast<uint16_t>(RecordType::AAAA) && rdlength == 16) {
    char ip_str[INET6_ADDRSTRLEN];
    struct in6_addr addr;
    memcpy(&addr, &response[offset], 16);  // ❌ UNCHECKED: offset + 16 may exceed buffer
    inet_ntop(AF_INET6, &addr, ip_str, sizeof(ip_str));
    result.addresses.push_back(ip_str);
    result.type = RecordType::AAAA;
}
```

**Also at lines 957, 1455** (async and DoH3 paths).

**Exploit Scenario**:
1. Attacker crafts malicious DNS response with `rdlength == 16` but actual data < 16 bytes
2. `offset + rdlength` exceeds `response.size()`
3. `memcpy` reads 16 bytes from out-of-bounds memory
4. **Information leak**: Adjacent heap memory exposed to attacker

**Root Cause**: Check `if (offset + rdlength > response.size()) break;` exists but is **after** the AAAA parsing block, not before.

**Fix**:
```cpp
// R13-C01: Validate bounds BEFORE memcpy
if (offset + rdlength > response.size()) break;  // Move BEFORE all parsing

if (rtype == static_cast<uint16_t>(RecordType::A) && rdlength == 4) {
    if (offset + 4 > response.size()) break;  // Extra safety
    char ip_str[INET_ADDRSTRLEN];
    snprintf(ip_str, sizeof(ip_str), "%d.%d.%d.%d",
             response[offset], response[offset + 1],
             response[offset + 2], response[offset + 3]);
    result.addresses.push_back(ip_str);
    result.type = RecordType::A;
} else if (rtype == static_cast<uint16_t>(RecordType::AAAA) && rdlength == 16) {
    if (offset + 16 > response.size()) break;  // Extra safety
    char ip_str[INET6_ADDRSTRLEN];
    struct in6_addr addr;
    memcpy(&addr, &response[offset], 16);  // ✅ Now safe
    inet_ntop(AF_INET6, &addr, ip_str, sizeof(ip_str));
    result.addresses.push_back(ip_str);
    result.type = RecordType::AAAA;
}
```

**Impact**: Remote attackers can leak heap memory contents via crafted DNS responses.

### 🔴 R13-C02: TLS Extension Parsing — Integer Overflow

| Property | Value |
|----------|-------|
| **File** | `src/core/src/ncp_dpi.cpp` |
| **Lines** | 131-145 |
| **Severity** | 🔴 **CRITICAL** (CWE-190: Integer Overflow) |
| **CVSS** | 8.1 (High) |
| **Status** | ⬜ Open |

**Vulnerable Code**:
```cpp
// ncp_dpi.cpp:131-145 — find_sni_hostname_offset()
uint16_t extensions_len = (static_cast<uint16_t>(data[pos]) << 8) |
                          static_cast<uint16_t>(data[pos + 1]);
pos += 2;

// R9-H14: Validate extensions_len doesn't overflow
size_t exts_end = pos + extensions_len;
if (exts_end > len || exts_end < pos) {  // Check for overflow
    exts_end = len;  // Clamp to buffer end
}
```

**Exploit Scenario**:
1. Attacker sends TLS ClientHello with `extensions_len = 0xFFFF`
2. `pos` is near end of buffer (e.g., `pos = len - 10`)
3. `exts_end = pos + 0xFFFF` wraps around (integer overflow)
4. `exts_end < pos` check triggers, but **clamping to `len` is wrong**
5. Parser continues reading beyond actual extension data
6. **Buffer over-read**: Adjacent memory parsed as extensions

**Root Cause**: The overflow check `exts_end < pos` is correct, but clamping to `len` doesn't fix the underlying issue — the parser should **reject** malformed packets, not clamp.

**Fix**:
```cpp
// R13-C02: Reject malformed extensions_len, don't clamp
uint16_t extensions_len = (static_cast<uint16_t>(data[pos]) << 8) |
                          static_cast<uint16_t>(data[pos + 1]);
pos += 2;

// Validate extensions_len doesn't exceed buffer
size_t exts_end = pos + extensions_len;
if (exts_end > len || exts_end < pos) {  // Overflow or out-of-bounds
    return -1;  // ❌ Reject malformed packet (don't clamp!)
}

while (pos + 4 <= exts_end) {
    // ... safe parsing ...
}
```

**Impact**: Crafted TLS ClientHello can cause buffer over-read, potentially leaking sensitive memory or causing crashes.

### 🔴 R13-C03: Thread Tracking — Use-After-Free Risk

| Property | Value |
|----------|-------|
| **File** | `src/core/src/ncp_dpi.cpp` |
| **Lines** | 2695-2702 |
| **Severity** | 🔴 **CRITICAL** (CWE-416: Use After Free) |
| **CVSS** | 7.8 (High) |
| **Status** | ⬜ Open |

**Vulnerable Code**:
```cpp
// ncp_dpi.cpp:2695-2702 — DPIBypass::stop()
// TODO: Replace with proper tracking vector<std::thread> active_connections_
// and join each thread explicitly for a complete fix.
{
    int spin = 0;
    while (impl_->active_connections_.load() > 0 && spin < 50) {
        std::this_thread::sleep_for(std::chrono::milliseconds(10));
        ++spin;
    }
}
// ❌ After 500ms timeout, destructor continues even if threads still running
// ❌ Impl members destroyed while threads may still access them
```

**Exploit Scenario**:
1. DPI bypass active with many concurrent connections
2. User calls `stop()` during high load
3. Thread pool shutdown takes >500ms (possible under load)
4. Spin-wait times out, destructor continues
5. `impl_` destroyed while proxy connection threads still running
6. **Use-after-free**: Threads access destroyed `impl_` members
7. **Crash or memory corruption**

**Root Cause**: Spin-wait with timeout is a workaround, not a proper synchronization mechanism. No guarantee threads have completed.

**Fix**:
```cpp
// R13-C03: Proper thread tracking with explicit joins
class DPIBypass::Impl {
    // ... existing members ...
    std::vector<std::thread> active_threads;  // ✅ Track all threads
    std::mutex thread_mutex;                   // ✅ Protect vector
};

// In handle_proxy_connection():
{
    std::lock_guard<std::mutex> lock(impl_->thread_mutex);
    impl_->active_threads.emplace_back([=]() {
        // ... proxy connection logic ...
        {
            std::lock_guard<std::mutex> lock(impl_->thread_mutex);
            // Remove self from vector on exit
            impl_->active_threads.erase(
                std::remove_if(impl_->active_threads.begin(),
                              impl_->active_threads.end(),
                              [](std::thread& t) {
                                  return !t.joinable();
                              }),
                impl_->active_threads.end());
        }
    });
}

// In stop():
{
    std::lock_guard<std::mutex> lock(impl_->thread_mutex);
    for (auto& t : impl_->active_threads) {
        if (t.joinable()) {
            t.join();  // ✅ Wait for all threads to complete
        }
    }
    impl_->active_threads.clear();
}
```

**Impact**: Crash or memory corruption during shutdown under high load. Potential for remote code execution if attacker can trigger shutdown during active connections.

### 🟠 R13-H01: DNS Compression Pointer — Infinite Loop Risk

| Property | Value |
|----------|-------|
| **File** | `src/core/src/doh.cpp` |
| **Lines** | 303-320 |
| **Severity** | 🟠 **HIGH** (CWE-835: Infinite Loop) |
| **Status** | ⬜ Open |

**Issue**: CNAME parsing has depth limit (`compression_depth < 16`), but main DNS parsing loop (lines 288-300) does **not** limit compression pointer following.

**Vulnerable Code**:
```cpp
// doh.cpp:288-300 — Question section parsing
for (int i = 0; i < qdcount && offset < response.size(); ++i) {
    while (offset < response.size() && response[offset] != 0) {
        if ((response[offset] & 0xC0) == 0xC0) {
            offset += 2;  // ❌ No depth limit!
            break;
        }
        offset += response[offset] + 1;
    }
    // ...
}
```

**Exploit**: Crafted DNS response with circular compression pointers causes infinite loop → DoS.

**Fix**: Add depth counter to all compression pointer following:
```cpp
int ptr_depth = 0;
while (offset < response.size() && response[offset] != 0) {
    if ((response[offset] & 0xC0) == 0xC0) {
        if (++ptr_depth > MAX_COMPRESSION_DEPTH) break;  // ✅ Limit
        offset += 2;
        break;
    }
    offset += response[offset] + 1;
}
```

### 🟠 R13-H03: Hex Decoder — Incorrect libsodium Usage

| Property | Value |
|----------|-------|
| **File** | `src/core/src/mimicry.cpp` |
| **Lines** | 68-85 |
| **Severity** | 🟠 **HIGH** (CWE-676: Unsafe Function Call) |
| **Status** | ⬜ Open |

**Vulnerable Code**:
```cpp
// mimicry.cpp:68-85
static bool safe_hex_decode_byte(uint8_t hi, uint8_t lo, uint8_t& out) {
    char hex[3] = {static_cast<char>(hi), static_cast<char>(lo), '\0'};
    uint8_t decoded;
    if (sodium_hex_decode(&decoded, hex, 2) == 0) {  // ❌ Wrong API!
        out = decoded;
        return true;
    }
    return false;
}

static unsigned int safe_hex_to_uint(const char* hex, size_t len) {
    unsigned int result = 0;
    for (size_t i = 0; i < len; ++i) {
        uint8_t decoded;
        if (sodium_hex_decode(&decoded, hex + i, 1) != 0) return 0;  // ❌ Wrong!
        result = (result << 4) | static_cast<unsigned int>(decoded);
    }
    return result;
}
```

**Issue**: `sodium_hex_decode()` expects **hex string length**, not number of bytes to decode. Calling with `len=1` decodes **half a byte** (nonsensical).

**Correct API**:
```cpp
// Decode "4A" (2 hex chars) → 0x4A (1 byte)
sodium_hex_decode(&out, "4A", 2);  // 2 = length of hex string

// Decode single hex char 'A' → 0x0A (not supported directly)
// Must pad: "0A" → 0x0A
```

**Fix**:
```cpp
static int hex_char_to_nibble(uint8_t c) {
    if (c >= '0' && c <= '9') return c - '0';
    if (c >= 'a' && c <= 'f') return c - 'a' + 10;
    if (c >= 'A' && c <= 'F') return c - 'A' + 10;
    return -1;
}

static bool safe_hex_decode_byte(uint8_t hi, uint8_t lo, uint8_t& out) {
    int h = hex_char_to_nibble(hi);
    int l = hex_char_to_nibble(lo);
    if (h < 0 || l < 0) return false;
    out = static_cast<uint8_t>((h << 4) | l);
    return true;
}

static unsigned int safe_hex_to_uint(const char* hex, size_t len) {
    unsigned int result = 0;
    for (size_t i = 0; i < len; ++i) {
        int n = hex_char_to_nibble(static_cast<uint8_t>(hex[i]));
        if (n < 0) return 0;
        result = (result << 4) | static_cast<unsigned int>(n);
    }
    return result;
}
```

**Impact**: Incorrect hex decoding may produce wrong values, breaking DNS label parsing.

### 🟠 R13-H07: Memory Region Overwrite — Wrong Pass Count

| Property | Value |
|----------|-------|
| **File** | `src/core/src/ncp_paranoid.cpp` |
| **Lines** | 1633-1680 |
| **Severity** | 🟠 **HIGH** (CWE-676: Unsafe Function Call) |
| **Status** | ⬜ Open |

**Issue**: `shred_file()` uses fixed 3-pattern overwrite (0x00, 0xFF, random), but `passes` parameter ignored.

**Vulnerable Code**:
```cpp
// ncp_paranoid.cpp:1633-1680
```

```cpp
    // Verify pass: read and check all bytes are 0x00
```
